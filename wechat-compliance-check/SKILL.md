# 微信公众号文章合规检测

## 概述

本 skill 对微信公众号文章内容进行发布前的合规风险检测，采用双层检测架构：

1. **词库扫描层**：基于本地词库 + DFA 算法进行高效关键词匹配，覆盖 10 大风险类别，支持全半角/繁简/拼音首字母等防绕过检测
2. **规则审查层**：对照微信官方运营规范逐条检查，覆盖内容层面（9 条红线）和行为层面（诱导、导流、资质、功能滥用等），引用官方原文条款和违规案例，输出结构化合规报告

**词库不内置在 skill 中**，首次使用时自动从开源仓库全量下载到本地，后续检测完全离线。

## 风险类别与优先级

| 优先级 | 类别 | 文件 | 说明 |
|--------|------|------|------|
| P0-致命 | 政治敏感 | `political.txt` | 领导人、敏感事件、分裂言论等 |
| P0-致命 | 法律法规 | `legal.txt` | 违法违规、涉黑涉恶、非法物品 |
| P1-严重 | 色情低俗 | `porn.txt` | 色情、软色情、低俗擦边 |
| P1-严重 | 暴恐血腥 | `violence.txt` | 暴力、恐怖主义、血腥描述 |
| P1-严重 | 毒品相关 | `drugs.txt` | 毒品名称、制毒方法、贩毒信息 |
| P1-严重 | 赌博相关 | `gambling.txt` | 赌博平台、博彩推广 |
| P2-警告 | 广告法违禁 | `advertising.txt` | 极限用语、虚假宣传、医疗用语 |
| P2-警告 | 虚假营销 | `fraud.txt` | 诱导分享、标题党、虚假承诺 |
| P3-提示 | 低质灌水 | `lowquality.txt` | AI 模板化内容、关键词堆砌 |
| P3-提示 | 短视频违禁 | `shortvideo.txt` | 抖音/短视频平台通用违禁词 |

用户可通过 `custom.txt` 添加自定义检测词，通过 `whitelist.txt` 添加白名单豁免词。

## 检测流程

### 输入格式

用户可提供任意组合：
- 文章标题
- 封面图描述（或图片）
- 正文内容（全文或片段）
- 配图描述（可选）
- 最近 3-5 篇已发布文章的标题和大致结构（可选，用于同质化检测）
- 账号定位描述（可选，用于整体风险评估）

支持的输入方式：
- 用户直接粘贴文本
- 用户提供 Markdown / .docx / .txt 文件路径
- 用户提供的 URL（用 WebFetch 获取）

### 第一步：运行 DFA 词库扫描

执行检测脚本（首次使用会自动下载词库，需联网）：

```bash
python3 {skill_dir}/scripts/detect.py \
  --text "文章内容" \
  --dicts {skill_dir}/dicts/
```

或从文件读取：

```bash
python3 {skill_dir}/scripts/detect.py \
  --file /path/to/article.txt \
  --dicts {skill_dir}/dicts/
```

输出格式为 JSON（添加 `--json` 参数），结构如下：

```json
{
  "summary": {
    "total_hits": 12,
    "risk_level": "high",
    "categories_hit": 4,
    "p0_fatal": 2,
    "p1_severe": 5,
    "p2_warning": 3,
    "p3_info": 2,
    "pass": false
  },
  "details": [
    {
      "category": "political",
      "priority": "P0-致命",
      "word": "xxx",
      "position": 120,
      "context": "...前后文各20字...",
      "suggestion": "建议删除或替换"
    }
  ],
  "article_preview": "标注了风险词的文章预览"
}
```

### 第二步：结构化规则审查

在词库扫描基础上，按以下 7 个模块逐一对照规则参考文档进行深度审查。每个模块需读取对应的 reference 文件，不能凭记忆判断。

**模块 1：标题检测**

读取 `reference/title-rules.md` 检测：
- 标题是否夸大或隐藏关键信息
- 标题与正文内容是否一致
- 是否存在诱导点击
- 是否包含敏感词扫描命中的词
- **新增**：是否存在煽动群体对立的表述（性别/地域/阶层标签化）
- **新增**：是否冒充权威来源（"央视曝光""官方紧急通知"等）

**模块 2：封面图检测**（如用户提供了封面图描述）

读取 `reference/image-rules.md` 中封面图部分检测：
- 封面图是否合规
- 是否包含敏感信息或误导性图片
- AIGC 生成图片是否标注

**模块 3：正文配图检测**（如用户提供了配图描述）

读取 `reference/image-rules.md` 中正文配图部分检测：
- 配图是否有真实截图（非 AI 生成假图）作为产品展示
- 截图中的敏感信息是否脱敏
- AIGC 图片是否为主体且无增量信息（属于低价值）

**模块 4：正文内容检测**

读取 `reference/content-rules.md` 检测：
- 政治敏感、虚假信息、色情暴力、涉医涉法等内容违规
- 数据、案例、截图是否真实
- 引用外部内容是否标注来源（不超过全文 20%）
- AI 生成内容是否经过充分人工改写
- **新增**：文中使用的地图截图是否标注正确（国界/领土）
- **新增**：是否存在煽动群体对立的表述

**模块 5：营销与广告表达检测**

读取 `reference/marketing-rules.md` 检测：
- 广告法违规词、极限词
- 诱导关注/分享/导流行为
- 正文是否推广产品/服务、介绍价格
- 是否容易被理解为"想卖东西"而非"分享作品"
- **新增**：保健品/健康产品是否夸大功效（对照 `platform-cases.md` §4.3 案例）

**模块 6：行为合规检测**

读取 `reference/behavior-rules.md` 检测：
- **诱导行为**：是否有诱导分享/关注/点击的 CTA 表述？（关键词匹配 + 语义分析）
- **违规导流**：是否嵌入微信号/二维码/外链引导至第三方平台？（关键词匹配 + 模式识别）
- **资质风险**：涉及金融/医疗/法律等领域是否缺少资质声明或免责声明？
- **功能滥用**：是否暗示可声明原创但实际不符合条件？
- **AI 合规**：AI 生成内容是否已标识？是否存在模板化批量发布风险？

**模块 7：综合风险评估（红线逐条审查）**

读取以下文件进行 9 条红线逐条审查：
- `reference/platform-rules.md` — 9 条红线官方规范全文（**主要判定依据**）
- `reference/platform-rules-basic.md` — 基础运营规范补充（覆盖诱导行为、滥用原创、信息标注来源、处罚机制等）
- `reference/platform-cases.md` — 违规官方案例（覆盖低创作度、诱导违规、违规导流、过度营销、标题煽动夸大 5 大类）

按风险从高到低顺序检查：

1. **红线 7 · 低创作度内容**（最常见风险）
   - 7.1 同质化：与最近几篇比，结构是否雷同？标题和封面是否产生系列模板感？
   - 7.2 抄袭搬运：大段引用是否标注来源？有无洗稿痕迹？
   - 7.3 信息量不足：是否有 ≥3 处具体数字、≥2 处独有细节、≥1 处真实经验？
   - 7.4 低价值 AIGC：AI 生成部分是否经过人工改写？配图有没有 AI 生成图充当证据？
   - **补充**（基础运营规范 3.4.3）：是否滥用原创声明？（直接贴 AI 原文不可声明原创）
   - **案例参照**：`platform-cases.md` §1 低创作度案例

2. **红线 5 · 不良信息**
   - 5.1 不规范医疗科普：涉及健康/心理但未标注"不替代专业医疗"？
   - 5.2 伪科学/封建迷信：涉及历史解读但未注明"仅供阅读参考"？
   - 5.3 标题问题：夸大、隐藏关键信息、标题与内容不符？
   - 5.4 导流内容：嵌入二维码、外链、联系方式？
   - **补充**（行为合规模块 6）：是否有诱导分享/关注/点击/导流？
   - **案例参照**：`platform-cases.md` §2 诱导违规、§3 违规导流、§5 标题煽动夸大

3. **红线 6 · 营销推广**
   - 正文是否推广产品/服务？是否介绍价格？
   - 是否容易被理解为"想卖东西"而非"分享作品"？
   - **案例参照**：`platform-cases.md` §4 过度营销

4. **红线 3 · 不实信息**
   - 数据、案例、运行结果是否真实？有无编造？
   - AIGC 图片是否虚构了不存在的功能或界面？
   - 标题是否冒充权威来源？

5. **红线 9 · 虚假人设**
   - 是否暗示了不存在的专业资质？
   - 配图中是否用了 AI 生成的虚假人像？

6. **红线 8 · 过时信息**
   - 引用的标准/法规/政策是否现行有效？

7. **红线 2 · 侵权**
   - 引用外部内容是否标注来源且不超过全文 20%？
   - **补充**（基础运营规范 4.24）：引用新闻/资讯是否标注来源？
   - **补充**（基础运营规范 4.17）：正文是否过于简短空洞？
   - **补充**（基础运营规范 4.14）：转载内容是否合规？

8. **红线 1 · 恶意使用**（通常不涉及，快速过）
9. **红线 4 · 公序良俗**（通常不涉及，快速过，除非内容敏感）

### 第三步：输出合规报告

综合词库扫描和规则审查结果，输出结构化报告：

```
# 合规检查报告

## 基本信息
- 文章标题：xxx
- 文章类型：xxx（作品展示/观点复盘/教程/其他）
- 检查时间：yyyy-mm-dd
- 词库扫描命中：x 条（P0: x, P1: x, P2: x, P3: x）

## 检测结果总览
- 红线通过项：x/9
- 行为合规检查项：x/x 通过
- 需关注项：x 条
- 违规项：x 条
- 综合判断：可发布 / 修改后发布 / 不建议发布

## 各模块检测结果

### [模块名]
**检测结果**：✅ 合规 / ⚠️ 低风险 / 🔴 高风险
**问题清单**（如有）：
- 问题描述
**风险说明**：平台可能采取的处理措施（参考处罚机制阶梯）
**优化建议**：
- 具体可操作的修改方向
**改写方案**（如有高风险内容）：
> [直接给出推荐替换文案]

（每个模块按此结构输出）

## 违规项（必须修改）

### 违规 #1：xxx
- **严重程度**：高/中/低
- **违反条款**：红线 X · XXX / 行为规范 §X.X
- **官方原文**：引用 platform-rules.md 或 platform-rules-basic.md 中的原句
- **对应案例**：如有匹配的官方案例，引用 platform-cases.md（如"参见§2.1案例1"）
- **文章中的具体问题**：引用文章原文段落，说明为什么违规
- **修改建议**：具体怎么改（只给方向，不改文章）

## 需关注项（建议修改）
（格式同上，严重程度标注为"关注"）

## 快速通过项
- 红线 1 · 恶意使用：不涉及
- 红线 4 · 公序良俗：不涉及
- 行为合规 · 资质风险：不涉及
- ...

## 综合发布建议
- **整体评级**：低风险 / 中等风险 / 高风险 / 严重违规
- **核心风险点**：（按严重程度排列）
- **发布建议**：可直接发布 / 建议修改后发布 / 强烈建议修改 / 不建议发布
```

### 第四步：提供修改建议

对 P0 和 P1 级别风险：
- 直接给出替换建议或删除建议
- 对政治敏感类，建议删除整段相关内容而非仅替换关键词
- 对广告法极限用语，提供常见替换方案（"最" → "优质"/"领先"/"出色"）
- 对诱导类 CTA，建议删除或改为内容驱动的表述
- 对导流类内容，建议移除联系方式/二维码/外链
- 只给修改方向和替换片段，不改写整篇文章

## 重要原则

1. **引用原文，不凭记忆**。每个违规项必须引用 `reference/` 下对应文件中的具体条款。如果不确定某条规定的原文，先回到参考文件中查。

2. **用案例说话**。如果 `reference/platform-cases.md` 中有类似的官方案例，必须引用。比如"这篇文章的结构问题，类似案例§1.1案例2中描述的'框架一致'问题"。

3. **区分"违规"和"需关注"**。只有明确触犯官方原文条款的才标为"违规项"。没有触犯但存在风险的是"需关注项"。不主观发挥。

4. **不下最终结论**。本工具是合规检查器，不是平台审核员。报告是给作者参考的，不是最终判决。

5. **本地检测，保护隐私**。所有检测在本地完成，不上传文章内容到任何外部服务。

6. **只检查，不改写**。不要把合规检查变成编辑修改。输出是一份报告，不是改后的文章。

## 特殊场景处理

### 涉及心理健康/医疗主题
- 重点检查红线 5.1：必须标注"不替代专业诊断/治疗"
- 检查有无"疗愈""治疗""改善症状"等医疗功效词
- 严重心理危机场景必须引导至 12356 等专业热线
- 检查是否涉及无资质行医（`behavior-rules.md` §4）

### 涉及历史/传统文化
- 重点检查红线 5.2：必须标注"仅供阅读参考"
- 检查有无"预言""算命""运势预测"等迷信框架
- 参考红线 3：历史内容不能有无依据揣测

### 涉及标准/法规引用
- 重点检查红线 8：引用版本是否现行有效
- 检查是否标注了版本号/年份

### 文章配图较多
- 检查是否有真实截图（非 AI 生成假图）作为产品展示
- 检查截图中的敏感信息是否脱敏
- 参考 `reference/platform-cases.md` §1.4 反面案例 2：AIGC 图片为主体无增量信息属于低价值

### 涉及产品/服务推荐
- 重点检查红线 6 营销推广 + 广告法
- 检查是否属于"纯广告内容"（参考 `platform-cases.md` §4.1 案例）
- 检查是否有虚假促销（虚构原价、虚假限时等）
- 参考 `reference/marketing-rules.md` 合规对照表

### 文末有 CTA（行动号召）
- 重点检查诱导行为（`behavior-rules.md` §1）
- 确认 CTA 不含诱导分享/关注/导流的表述
- 合规 CTA 应基于内容价值而非奖励驱动
- 参考 `reference/marketing-rules.md` §7 互动引导合规写法

### 涉及金融/投资/法律建议
- 检查资质要求（`behavior-rules.md` §4）
- 必须有免责声明："不构成投资建议""投资有风险""本文不构成法律意见"
- 不得推荐具体理财产品或给出具体投资建议

### 涉及地图/地理信息
- 检查地图使用是否规范（`platform-rules-basic.md` §4.19）
- 国界线、台湾/港澳标注是否正确
- 建议使用官方标准地图

## 词库管理

### 首次使用（自动）

首次执行检测时，脚本会自动检测词库是否存在。如果不存在，会自动执行初始化：
- 从 3 个开源仓库下载全量词条
- 整个过程约需 30-60 秒（取决于网络）
- 初始化完成后的检测完全离线

也可以手动触发初始化：

```bash
python3 {skill_dir}/scripts/update_dicts.py --init
```

### 增量更新词库

更新频率建议：每月一次，或手动触发。

## 防绕过检测

检测脚本内置以下防绕过处理：
- **全半角转换**：`ａ` → `a`，`１` → `1`
- **大小写归一**：统一转小写
- **繁体转简体**：基本繁简转换
- **特殊字符过滤**：移除词间插入的 `*`、`.`、`-`、空格等干扰字符
- **拼音首字母**：常见拼音首字母缩写识别（如 `xjp`）
- **数字替换**：`0` ↔ `〇`，`1` ↔ `一` 等

## 局限性说明

- 本工具基于公开规则和词库进行风险评估，**无法保证 100% 通过微信审核**
- 微信平台采用人工 + AI 双重审核，且规则持续更新，存在误判可能
- 政治敏感内容判断依赖上下文，词库只能做关键词提示
- 图片内容检测仅基于用户文字描述，无法扫描实际图片像素内容
- 词库无法覆盖所有可能的违规内容，检测结果仅供参考
- P2/P3 级别需人工判断，部分词汇在特定语境下合规
- P0 政治敏感类建议宁可误杀也不漏放
- AI 内容标识：根据 2025 年微信新规，AI 生成内容必须标识
- 行为规范层面的检测主要基于关键词匹配和语义分析，复杂的上下文判断仍需人工复核
- 如需确保合规，建议参考微信官方最新公告并进行人工复核
